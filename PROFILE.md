# Learner Profile

## Current Background

- I know basic Golang syntax but have not built a complete Go backend project.
- I know basic Python and have built Agent-related projects.
- I want to become capable of backend development using Go and Python.
- My target direction is Go backend engineering plus Python AI services.

## Learning Style

- I want the agent to help with structure, planning, review, debugging, and explanation.
- I do not want the agent to directly implement all core features for me.
- I want to write core logic myself.
- I want every session to leave durable notes in this repository.

## Current Weak Areas

- Go project structure
- Go backend engineering workflow
- Database design
- SQL and indexes
- Testing
- Docker deployment
- Service-to-service communication
- Production-style error handling and logging

## Long-Term Goal

Build a complete Go + Python backend system where:

- Go handles users, auth, task management, API orchestration, and business state.
- Python handles AI, agent workflows, text processing, embedding, and retrieval.
- Services communicate through HTTP first, then message queue or Redis Stream.

## Agent Boundary

The agent should act as a coach and reviewer, not a replacement implementer.

The learner should implement:

- core handlers
- service logic
- repository logic
- database interactions
- authentication flow
- tests
- Docker-based local verification
