# Review: 2026-06-24 T011 User Login

## Result

Accepted after fixes. The implementation keeps a clean handler/service/repository shape, passes static checks, and satisfies the T011 acceptance criteria.

## Findings

1. Fixed after first review: login now uses `bcrypt.CompareHashAndPassword` against the stored password hash instead of generating a new hash and comparing strings.
2. Fixed after first review: invalid credentials now return `401 Unauthorized` with `invalid email or password` instead of `400 missing parameter`.
3. Fixed after first review: the password-related handler error branch now returns immediately after writing the response, avoiding duplicate responses.
